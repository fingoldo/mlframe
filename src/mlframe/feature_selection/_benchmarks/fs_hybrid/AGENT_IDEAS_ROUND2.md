Generated after round-1 (see AGENT_IDEAS.md). Each round-2 idea is grounded in a round-1 learning and avoids

## RFECV (round-2)
- R2r-1 Variance-aware permutation n_repeats early-stop — DONE-doc (deferred, speed-only): correctness-affecting (changes which repeats run) speedup on the shipped permutation default. Agents flagged the ABSOLUTE win is small on the small synthetics (p~30-110, fit dominates); real payoff is production-scale held-out folds. Accuracy-first mandate prioritises the AUC/robustness benches; this is a safe future speed lever (gate so it never changes the boundary decision). Not benched now (no accuracy change to measure on the AUC bed).
- R2r-2 Cross-seed/bootstrap support_ aggregation (frequency-vote final support_ across independent fits) — DONE-doc (stability lever, NOT AUC default): benched single-fit vs B=4 bootstrap frequency-vote, 3 scenarios x 4 outer seeds (round2_rfecv_crossseed_bench.py). AUC is a TIE (mean single 0.8117 vs agg 0.8112, -0.0005). Cross-seed AUC std barely moves (0.0101->0.0090). The real, large effect is SELECTION reproducibility: mean pairwise Jaccard 0.729->0.925, driven by highnoise 0.374->0.887. So at 4x fit cost it buys reproducible feature sets, not better downstream AUC. Per CLAUDE.md §6 (equal accuracy -> fastest wins) single-fit stays the default; aggregation is the right primitive for HybridSelector's SHARED support and for any reproducibility-critical use. Same precision/stability category as hetero_vote.
- R2r-3 Batched-predict permutation — DONE-doc (deferred, speed-only): bit-identical compute restructuring. Agent flagged the win EVAPORATES when model.fit dominates predict, which it does on the 5k-row synthetics; payoff is wide production folds. Zero-accuracy-risk future lever; deferred (nothing to measure on the AUC bed; sklearn's permutation_importance is already vectorised per-column).
- R2r-4 Data-shape + step-1 rank-agreement routing impurity<->permutation (replace flat 4M cap) — DONE-doc (no measurable effect on the bed; scale-only routing): the flat 4M-cell cap routes to impurity only when permutation is too costly; on the synthetic (cells well under 4M) permutation ALWAYS runs, so rank-agreement routing changes nothing measurable here and only matters at production scale. Permutation already won the round-1 default shootout (10/12). The flat cap is a safe conservative default; rank-agreement routing is a future scale-side refinement with a calibration-revert guard. Same disposition class as R2r-1/3/7 (speed/scale, not bed-accuracy).
- R2r-5 Raw-corr cluster collapse as SEARCH-SPACE reduction (rep + free slots, re-expand) — DONE-doc (RFE already prunes redundancy; mirrors R2b-6): RFECV already eliminates redundant copies via CV-ranked backward elimination, so pre-collapsing correlated features mainly saves search time and re-expansion restores the same information -> AUC-neutral, the same outcome being measured for the Boruta analog R2b-6. The flagged B-extra-collapse risk (collapsing weakly-correlated distinct signal) is real on the monotone/weakmix cells. Speed/search reduction, not an accuracy default; revisit as a shared-cluster reuse in HybridSelector where the clustering is computed once anyway.
- R2r-6 Plateau-onset n_features rule — DONE-shipped-as-OPTION (not default): implemented as n_features_selection_rule='plateau' + validators + test. Benched off cv_results_ across 6 scenarios: wins 0/6 on downstream AUC - behaves like one_se_min (same N in 4/6: base 18, manyredundant 15, weakmix 10), over-prunes the noise-robust regimes where one_se_max wins (highnoise 0.795 vs 0.814; manyredundant 0.786 vs 0.823). Agent's predicted 'collapses toward one_se_min on flat GBM tails' confirmed. Kept as an explicit parsimony-oriented option (distinct on rise-then-plateau curves), NOT the default.
- R2r-7 Early-stopped/reduced-fold interior CV — DONE-doc (deferred, speed-only): changes the MBH search trajectory (risk) for a refit-cost win that is small on small synthetics. Deferred behind the accuracy benches; revisit at production scale with the calibration-revert guard.
- R2r-8 Permutation memoisation across RFE steps — DONE-rejected (grounded): MBH revisits the same N with DIFFERENT subsets (documented F9 loser-rollback behaviour), so exact (subset,seed) cache collisions are near-zero -> the cache would have ~0 hit rate. Agent's own gate (collision<5% -> reject) is met by the code's design. Also intersects the 'runtime caches break pickle' rule. Not worth implementing.

## BorutaShap (round-2)
- R2b-1 Wire hetero_vote + test stack — DONE-doc (benched 6x2): hetero_vote is a PRECISION/parsimony tool, NOT an AUC default. It drives noise to 0 (vs boruta 1.6) + compact (7-9 vs 17 feats) but UNDER-RECOVERS weak signal -> mean AUC 0.742(hetero2)/0.736(hetero5) < boruta 0.764, wins only 2/12. Round-1's 5/7-0/32 was one lucky scenario. Kept as the precision option (docstring corrected); plain boruta wins for downstream AUC. AND/OR-stack-with-stability moot (hetero already over-prunes recall).
- R2b-2 Cheapen hetero_vote n_shadow_trials — DONE-shipped: benched 2 vs 5 -> n_shadow_trials=2 marginally BEATS 5 (0.742 vs 0.736 mean) at 2.5x speed (cross-MODEL is the mechanism, not cross-trial). Lowered module default 5 -> 3 (robust middle).
- R2b-3 Shadow-null-calibrated vote-fraction threshold (non-parametric panel-agreement null) — DONE-doc (no AUC headroom + panel too small): a non-parametric panel-agreement null needs many panel members to estimate the vote-fraction distribution; the default 3-model panel cannot support it. More fundamentally hetero_vote's deficit is RECALL not precision (R2b-1: accepted-noise already ~0), and a principled null only moves the precision threshold, so it cannot close the AUC gap to boruta. Refinement of a non-default precision tool; deferred to that niche.
- R2b-4 CV-skill-weighted vote (downweight structurally-blind panel member: linear on monotone) — DONE-benched (rejected, premise falsified): implemented hetero_vote(weight_by_cv_skill=True) + round2_hetero_skillweight_bench.py (6 scenarios x 2 seeds). Result: skill-weighting changed the selection in 0/12 cells (mean AUC identical to equal-weight, 0.7418; both trail boruta 0.7636). The mechanism's premise is FALSE on this bed: every panel member keeps a balanced CV skill (~0.20-0.29) even on monotone/weakmix (the monotone signal is largely linearly recoverable, so the linear member is NOT near-chance), so there is no member to downweight and weighting ~= equal. hetero_vote's recall gap is structural to the cross-model agreement requirement, not one blind voter. Option kept off-by-default + honestly documented (not a recall fix); boruta stays the AUC default, hetero_vote stays the equal-weight precision tool.
- R2b-5 Drop per-trial SHAP from the Boruta loop; gini/permutation gate driver — DONE-shipped (default flipped Shap->gini; permutation added): benched gini vs Shap vs permutation-held-out over the 6x2 bed (round2_boruta_driver_bench.py). SHAP is DOMINATED on both axes: worst mean AUC 0.755 AND ~137x slower than gini (20-30 min/fit, per-trial SHAP recompute). gini 0.762 @ ~11s; permutation-held-out 0.766 @ ~117s AND drives accepted-noise to ~0 in 10/12 cells (gini 1-5, SHAP up to 11). Per CLAUDE.md S6 the +0.0035 perm-over-gini is within cross-seed noise -> equal accuracy -> fastest wins: SHIPPED default importance_measure "Shap" -> "gini". Permutation is the new accuracy+precision option (added this session per user; honest held-out via the existing 30% split, permutation_n_repeats param, 5 unit tests). Class docstring DRIVER SELECTION block records all three.
- R2b-6 Pre-merge raw-corr clusters before the shadow gate, re-expand after accept (manyredundant) — DONE-confirmed-win (-> HybridSelector): premerge_gini WON 7/12 cells overall and is fast (~10s), dramatically best on redundant (mean 0.798 vs gini 0.78, recovers 7/7 base vs 6/7) and monotone (0.809). Collapsing raw-corr clusters to one rep before the gini gate de-dilutes the shadow comparison; re-expanding accepted reps lets the downstream exploit the cluster. It is FREE for a caller that already computes clustering, so it is implemented in the HybridSelector (compute-once clusters) rather than bolted onto the concurrently-edited BorutaShap class; documented in the BorutaShap DRIVER SELECTION block.
- R2b-7 Integrate panel vote INTO Boruta in-loop gate (cross-model + iterative confirmation) — DONE-doc (rejected for AUC; conditional gate failed): explicitly conditional on R2b-1, which returned hetero_vote = precision-tool-not-AUC-default. Folding the cross-model panel into the in-loop gate inherits the same recall deficit at L-effort cost with no AUC payoff; the standalone hetero_vote already serves the precision/parsimony niche. Not worth the L implementation.
- R2b-8 OOF/drop-column importance + panel — DONE-rejected (grounded): hetero_vote (the panel) already drives accepted-noise to ~0 (R2b-1) - its problem is RECALL, not noise, so adding OOF importance (which targets the noise leak) has no headroom. And same-draw OOF inherits draw-level spurious (B5/B8 findings). No live problem for it to solve.

## ShapProxiedFS (round-2)
- R2s-3 Refine net-value ablation: refine=False top20/top40 vs refine=True tol0.02/tol0.005 — DONE-shipped (default flipped): benched 4 configs x 3 over-prune scenarios x 2 seeds (round2_shap_bench.py). within_cluster_refine is NET-POSITIVE (refine=False won 0/6 cells). The over-pruning was the LOOSE tol, not refinement: tol=0.02 over-prunes (~6 feats, 4.3/7 informative, mean AUC 0.792, catastrophic on manyredundant sd0=0.7745) while tol=0.005 is the accuracy optimum (~11 feats, 5.0/7, mean AUC 0.795, best downstream LightGBM 0.808, no catastrophe, beats refine=False in 4/6 cells). SHIPPED: flipped ShapProxiedFS class default parsimony_tol 0.02 -> 0.005 (refine stays True); fs_hybrid ShapSel wrapper now uses it; regression test test_shap_proxied_parsimony_default_is_measured_optimum locks it.
- R2s-1 Holdout-SPLIT-variance-calibrated parsimony band (true 1-SE on real noise, not model-seed std~0) — DONE-doc (subsumed by R2s-3 fixed-tol win): the confirmed S4 root cause (band mis-scaled vs noise) is fixed by the measured fixed tol=0.005, which already removes the catastrophic over-prune and tops the bed. An adaptive split-variance band adds machinery + per-fit variance estimation cost for at most the residual gap above a tol that already wins 4/6; CLAUDE.md §6 (fastest at equal accuracy) favours the constant. Revisit only if a future dataset shows the fixed tol mis-scaled (the band would then be the named fix).
- R2s-4 Absolute-floor parsimony guard (never drop a feature worth > delta_abs honest loss) — DONE-doc (this IS parsimony_tol): parsimony_tol=0.005 is precisely an absolute honest-loss floor (drop a member only while honest-loss increase < tol). R2s-3 calibrated that floor to the measured optimum; a second floor knob is redundant.
- R2s-2 Selection averaged over K holdout splits (reuse phi; vote inclusion) — DONE-doc (stability lever, not AUC default): same variance-reduction class as R2r-2, which benched AUC-NEUTRAL (tie) and stability-positive at K-fold cost. The honest enemy here is cross-seed variance, but averaging reduces SELECTION variance (reproducibility), not mean downstream AUC, so it is not an accuracy default. It is the natural primitive for HybridSelector's shared, reproducible support; R2s-7 (OOF-SHAP cache) is its speed enabler if ever shipped.
- R2s-5 Skill-normalized selection band ((loss - random_baseline), not raw brier) — DONE-doc (subsumed): a rescaling of the same parsimony band R2s-3 calibrated; on weakmix the denom (loss - random_baseline) -> ~0 and the band explodes (the clamp caveat in the idea), reintroducing over-pruning the fixed 0.005 tol avoids. No headroom above the shipped fixed tol.
- R2s-6 Per-feature cross-fold phi-stability as DROP-PROTECTION (distinct from S3/S6) — TODO (M, risk: copies also stable)
- R2s-7 Cache OOF-SHAP across splits — DONE-doc (deferred, conditional): pure speed ENABLER for R2s-2 (K-split averaging). Ship only if R2s-2 proves a robustness win (benched separately). Cache infra already exists (cache_dir); deferred until R2s-2's verdict. Intersects 'runtime caches break pickle' (needs __getstate__ exclusion if added).
- NOTE: R2s-1/R2s-4/R2s-5 are three formulations of ONE fix (parsimony band mis-scaled vs noise) -> bench together as a 2x2x2 calibration; R2s-3 gates the cluster.
