## RFECV
- R1 SHAP-interaction / H-stat synergy bundling — DONE-rejected (cheap-test): SHAP-interaction surfaces the TRUE operand pair only when interactions dominate (xor2: (inf_4,inf_5) is the #1 pair) but is SWAMPED by spurious pairs otherwise - base: true pair ranks 10 (top pairs are independent-linear inf_1-inf_2 and even noise_22-inf_1); manyredundant: rank 56. Bundling the top-k interaction pairs would mostly bundle spurious/redundant pairs -> unreliable, likely harmful. Only viable on interaction-dominated data, which is exactly where MRMR-FE already constructs the product.
- R2 Paired re-add rescue pass at finalize — DONE-rejected (depends on R1): selecting WHICH dropped pairs to re-add needs the SHAP-interaction ranking, which R1's cheap-test showed mis-ranks the true pair (rank 10 base / 56 manyredundant) under spurious pairs. Re-adding the top-interaction dropped pairs would re-inject spurious/noise pairs. Same failure as R1.
- R3 OOF permutation as default tree importance — DONE-shipped-as-DEFAULT: shootout (6 scen x 2 seeds) permutation wins 10/12 lgbm cells, best mean AUC 0.7954, cleanest (2.5 vs 6.2 noise). 'auto' now routes to permutation when held-out target present + cells<=4M (cost gate), else impurity. SHAP tested = WORST (0.786) + slowest (153s) -> not default.
- R4 Auto conditional-permutation on correlated clusters — DONE-rejected (wide bench would-be; base: hurts, keeps 5 noise, lower AUC)
- R5 Per-downstream-model-class selection tailoring — DONE-rejected (grounded): the fs_hybrid benchmark already shows rfecv_logit (coef importance, tailored to the linear consumer) is the WORST RFECV variant (mean AUC 0.753) and underperforms rfecv_lgbm even on the LOGIT downstream - because linear-coef importance can't capture the interaction/quadratic the data needs; FE is the lever for linear models, not selection-tailoring (agent's own skeptical flag confirmed).
- R6 Stability-gated acquisition (penalize unstable N in MBH target) — DONE-rejected (grounded): stability_vs_n_curve_ is ~1.0 across ALL N on these small-p problems (observed in the pareto_front_ test: stability=1.0 for every front point), so there is NO stability gradient to gate the MBH acquisition target on -> no effect. Agent flagged value only on plateau-prone large-p (p>>n); out of scope for the current bench.
- R7 Multi-objective Pareto front (AUC, N, stability) + pareto_front_ — DONE-doc: shipped pareto_front_/pareto_knee_ as READ-ONLY DIAGNOSTICS + test. Benched as a SELECTION RULE -> REJECTED: knee lost 0/6 scenarios on downstream AUC (over-prunes to ~4-8 feats); argmax/one_se_max won 3/3 (validates current recall default). Knee is a parsimony-only diagnostic, not an accuracy default.
- R8 Warm-start MBH surrogate with MI/interaction ranking; MI computed OR injected externally (e.g. MRMR) — DONE-doc: cheap-test shows plain MI ranks the interaction operands AMONG/BELOW noise (base: ranks 44 & 50 of 51, median-noise 28; xor2: avg 21 vs noise 23), while quadratic/linear are top-ranked (already safe). So INTERNAL-MI warm-start cannot rescue the synergy operands (would eliminate them first, like the alphabetical cold-start) -> internal variant REJECTED. The cold-start IS alphabetical when FI history is empty (get_actual_features_ranking name-tiebreak), so the EXTERNAL-injection plumbing (feed MRMR's synergy-aware conditional-MI/JMIM ranking) is the only viable form; deferred (4-file MBH-threading change; its win is unverifiable on the synthetic without a synergy-aware external ranking, and MRMR internals are owned by a concurrent session). User-requested external-MI injection is the right API; implement when a synergy-aware ranking source is wired.
- R-imp Importance shootout impurity/permutation/shap across scenarios -> set best default + profile — DONE: drove R3 (permutation shipped as default, ca48ba82). permutation 10/12 wins, mean AUC 0.7954 vs impurity 0.7900 vs SHAP 0.7858. (Profiling/speed-up of permutation = follow-up optimisation per CLAUDE.md.)

## BorutaShap
- B1 Model-X knockoff shadows — DONE-rejected (cheap-test): Gaussian (equicorrelated) knockoff shadows CRUSH signal recovery (signal hit-rate 0.86->0.14) while zeroing noise (0.04->0.00). With redundant clusters Sigma is ill-conditioned -> s~0 -> knockoffs ~= the real features -> they absorb the reals' importance, so SIGNAL fails the gate too. Agent's predicted failure mode (needs well-conditioned Sigma / SDP+cvxpy). Reveals the core tension: a covariance-preserving shadow that hides spurious-noise structure also hides SIGNAL structure -> self-defeating for an all-relevant gate. (Permutation shadows give signal 0.86 / noise 0.04; the residual one-noise leak is draw-level, handled by the shipped cross-subsample stability.)
- B2 Per-cluster / conditional permutation shadows — DONE-rejected (grounded): the agent's own 'cheaper 80% of B1'. B1's cheap-test showed covariance-preserving shadows CRUSH signal (structure-preserving shadow hides signal too); the conditional/block form is the weaker member of that self-defeating family. And its goal (drop the model-specific noise leak) is already achieved by the SHIPPED B7 cross-model vote (noise 0/32) + cross-subsample stability. Redundant alternative to a solved problem.
- B3 SHAP interaction-value importance (catch inf_4) — DONE-rejected (grounded by R1 cheap-test): SHAP-interaction surfaces the true operand pair only in interaction-dominated data (xor2 rank 1) and is swamped by spurious pairs otherwise (base rank 10, manyredundant 56). Using it to catch inf_4 in Boruta would be similarly unreliable - promotes spurious pairs except where MRMR-FE already wins.
- B4 Gumbel/EVT shadow-max threshold — DONE-rejected (grounded): the documented leak beats even the MAX shadow ~58/60 (BorutaShap docstring) - the spurious importance is GENUINELY high (structure), not a threshold-calibration artifact, so a sharper EVT quantile cannot drop it. Agent's own likely-partial flag confirmed.
- B5 Independent-draw generalization gate — DONE-rejected (already tested): train_or_test='test' (a same-draw held-out split) was measured NOT to fix the leak - the draw-level spurious structure is in BOTH halves of one draw. A genuinely independent draw would resolve it but the selector doesn't have one within a single fit (the fundamental limit already documented in the class docstring).
- B6 SPRT per-feature early-stop — DONE-doc: the COARSE form (stop the whole loop when no feature is tentative) is already SHIPPED (5dca23ea, ~104/150 on clean data). Full per-feature SPRT (anytime-valid sequential test, stop each feature individually) is an incremental EFFICIENCY refinement only - marginal extra speedup over the shipped coarse early-term, and the dominant cost on noisy data is the perpetually-tentative features, which SPRT also cannot decide faster. Deferred as low-value-vs-effort.
- B7 Heterogeneous base-model importance ensemble — DONE-SHIPPED (the one Boruta idea that WORKS): new feature_selection/hetero_vote.py heterogeneous_relevance_vote() + test. Cross-model shadow vote (>=2/3 of tree/linear/distance) keeps signal 5/7 but drops noise 0/32 on base, vs single-tree leaking 2 noise (cross-model disagreement kills model-specific spurious). Cheaper/more robust noise-leak fix than 10x cross-subsample stability for the common case. (Misses the 2 interaction operands - universal marginal blindspot.) Standalone module to avoid conflict with concurrently-edited boruta_shap.py.
- B8 Permutation / drop-column (OOB) importance metric — DONE-doc: permutation importance is now VALIDATED as superior generally (RFECV default flip, shootout 10/12; and it is one of B7's panel importance extractors). For Boruta's SPECIFIC draw-level leak it is only PARTIAL - permutation-on-the-same-draw does not fix a draw-level spurious correlation (shown earlier via train_or_test='test'); the residual leak needs cross-model (B7) or cross-subsample (stability) voting. Recommendation: prefer permutation over gini for Boruta's gate where affordable (less impurity-bias), but pair with B7/stability for the draw-level leak.

- B-extra sklearn clone-ability — DONE-shipped fdd8028c
- B-extra redundancy-collapse post-accept — DONE-rejected (drops clean signal)

## ShapProxiedFS
- S1 Downstream-model honest loss — DONE-rejected: the model= variant (proxy+honest matched to LightGBM) was benched 3 seeds = high-variance WASH (+0.071/+0.006/-0.058, mean +0.007; seed0 win was luck). The separate-honest-model plumbing is dominated by the simpler, robust within_cluster_refine=False (335ec1e4) which already removes the over-pruning. Not worth the M-effort plumbing for a wash.
- S2 Interaction-deficit as bias-corrector regression feature — DONE-rejected (cheap-test falsified, s2_interaction_deficit_test.py): agent predicted corr(interaction_mass, honest-proxy) NEGATIVE (proxy under-credits synergy); measured POSITIVE +0.49 xor2 / +0.33 base. TreeSHAP folds interaction into MAIN-effect phi, so the additive proxy already absorbs synergy and OVER-credits interaction subsets -> the "down-correct to recover synergy" premise is backwards for a TreeSHAP proxy. (User favorite, but the falsifiable test the user liked did its job.)
- S3 Fidelity-adaptive search depth / refine aggressiveness — DONE-rejected (cheap-test): fidelity is HIGH across all regimes (xor2 0.83, base 0.91, manyredundant 0.94) including the WORST over-pruning (manyredundant 2/7 base @ fidelity 0.94). Fidelity measures proxy RANKING quality, not refine over-pruning -> decoupled, can't gate refine on it. (refine=False remains the over-pruning fix.)
- S4 Variance-calibrated parsimony_tol (from holdout-loss std) — DONE-doc: cheap-test CONFIRMS premise — fixed parsimony_tol=0.02 exceeds the drop-one brier contribution of 6/7 informative features (median ~0.013 < 0.02) on base + manyredundant, so refine over-prunes real signal. Documented on the param as the over-pruning dial (lower to ~0.005 or refine=False); the shipped within_cluster_refine=False (335ec1e4) already remedies it in the benchmark. Full variance-calibration left as a future refinement (LightGBM holdout brier is near-deterministic here, so a z*std tol would collapse to ~0 = keep-all; needs the holdout-SPLIT std, not model-seed std).
- S5 Membership->loss surrogate GBM on trust-guard anchors — DONE-rejected (grounded): its motivation (a surrogate that captures interaction the additive proxy misses) is dead - S2's cheap-test showed the TreeSHAP additive proxy OVER-credits interaction subsets (folds interaction into main effects), so there is no synergy under-crediting for a surrogate to recover; plus agent-flagged sample-starvation (~30 anchors over ~28 units can't learn pairwise structure).
- S6 SHAP-vector (attribution) clustering instead of raw-corr — DONE-rejected (cheap-test): phi-corr clustering merged 0/4 (manyredundant) and 0/1 (xor2) redundant groups vs raw-corr 4/4 & 1/1, and in xor2 even failed to keep interaction operands separate. Correlated copies receive arbitrary/split SHAP credit so their attribution vectors are NOT correlated -> phi-corr can't group redundants (the point of clustering). Raw-value correlation is the right redundancy metric; keep cluster_backend on values.
- S7 Permutation-shortcut refine (speed) — DONE-rejected (in-context): refine is disabled by recommendation (within_cluster_refine=False is the over-pruning fix), so speeding it is low value; and the agent flagged the permutation shortcut WORSENS over-pruning (permutation importance of a column with correlated survivors under-credits it -> more drops), the exact failure mode we avoid.
- S8 Pair-aware anchor sampling — DONE-rejected (grounded): its only consumers are S2 (corrector) + S3 (fidelity-adaptive depth), BOTH rejected. Pair-aware anchors only change the fidelity measurement, which S3 showed gates nothing useful (fidelity high even at 2/7 recovery). No live consumer -> moot.
