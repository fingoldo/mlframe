## ShapProxiedFS
- S1 Downstream-model honest loss — DONE-rejected: the model= variant (proxy+honest matched to LightGBM) was benched 3 seeds = high-variance WASH (+0.071/+0.006/-0.058, mean +0.007; seed0 win was luck). - S2 Interaction-deficit as bias-corrector regression feature — DONE-rejected (cheap-test falsified, s2_interaction_deficit_test.py): agent predicted corr(interaction_mass, honest-proxy) NEGATIVE (proxy under-credits synergy); measured POSITIVE +0.49 xor2 / +0.33 base. TreeSHAP folds interaction into MAIN-effect phi, so the additive proxy already absorbs synergy and OVER-credits interaction subsets -> the "down-correct to recover synergy" premise is backwards for a TreeSHAP proxy. (User favorite, but the falsifiable test the user liked did its job.)
- S3 Fidelity-adaptive search depth / refine aggressiveness — DONE-rejected (cheap-test): fidelity is HIGH across all regimes (xor2 0.83, base 0.91, manyredundant 0.94) including the WORST over-pruning (manyredundant 2/7 base @ fidelity 0.94). Fidelity measures proxy RANKING quality, not refine over-pruning -> decoupled, can't gate refine on it. (refine=False remains the over-pruning fix.)
- S4 Variance-calibrated parsimony_tol (from holdout-loss std) — DONE-doc: cheap-test CONFIRMS premise — fixed parsimony_tol=0.02 exceeds the drop-one brier contribution of 6/7 informative features (median ~0.013 < 0.02) on base + manyredundant, so refine over-prunes real signal. Documented on the param as the over-pruning dial (lower to ~0.005 or refine=False); the shipped within_cluster_refine=False (335ec1e4) already remedies it in the benchmark. Full variance-calibration left as a future refinement (LightGBM holdout brier is near-deterministic here, so a z*std tol would collapse to ~0 = keep-all; needs the holdout-SPLIT std, not model-seed std).
- S5 Membership->loss surrogate GBM on trust-guard anchors — DONE-rejected (grounded): its motivation (a surrogate that captures interaction the additive proxy misses) is dead - S2's cheap-test showed the TreeSHAP additive proxy OVER-credits interaction subsets (folds interaction into main effects), so there is no synergy under-crediting for a surrogate to recover; plus agent-flagged sample-starvation (~30 anchors over ~28 units can't learn pairwise structure).
- S6 SHAP-vector (attribution) clustering instead of raw-corr — DONE-rejected (cheap-test): phi-corr clustering merged 0/4 (manyredundant) and 0/1 (xor2) redundant groups vs raw-corr 4/4 & 1/1, and in xor2 even failed to keep interaction operands separate. Correlated copies receive arbitrary/split SHAP credit so their attribution vectors are NOT correlated -> phi-corr can't group redundants (the point of clustering). Raw-value correlation is the right redundancy metric; keep cluster_backend on values.
