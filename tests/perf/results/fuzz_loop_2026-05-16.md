## iter #1' — regression x cb,lgb 100K (seed=20260516)

**Combo:** regression, models=cb+lgb, n_rows=100K

**Baseline hotspot (wall=1.0s, status=CatBoostError GPU not found):**
`_cached_gpu_info` → `get_gpuutil_gpu_info` → GPUtil import → setuptools chain — **0.729s / 0.952s wall**. GPUtil pulls `_distutils_hack`, `setuptools`, distutils at first call.

**Optimization:** `src/mlframe/training/_cb_pool.py:465-485` — added `shutil.which("nvidia-smi")` pre-check before calling `get_gpuinfo_gpu_info()`. When nvidia-smi is absent (non-GPU CI / dev VMs / cloud workers without CUDA), the GPUtil import is fully skipped; cached as empty list.

**After (wall=0.2s, same status):** GPUtil import cost dropped from 0.631s to 0.058s. **Speedup unclear:** nvidia-smi IS on PATH on this dev machine (`C:\windows\system32\nvidia-smi.EXE`), so the pre-check falls through to `get_gpuinfo_gpu_info()`; observed wall delta is most likely OS file-cache warming between cold→warm runs. The fix remains valid (~0.7s/call on non-GPU machines, no-op on GPU machines).

## iter #2 — regression x lgb 300K (seed=2026051617)

**Combo:** regression, models=lgb (single), n_rows=300K

**Baseline hotspot (wall=5.0s, status=OK):**
- `lightgbm.basic.update` → 2.337s (library-bound, no fix)
- `lightgbm.engine.train` → 2.738s (library-bound)
- **`joblib.externals.loky.context._count_physical_cores_win32` → 1.457s** via `subprocess.run` → wmic. Called once but cumulatively dominates the suite wall. LightGBM's `_process_n_jobs` invokes `joblib.cpu_count()` when `n_jobs is None`, which on Windows shells out to wmic for physical-core counting.

**Optimization:** `src/mlframe/training/lgb_shim.py:441-462` — pre-fill `self.n_jobs = os.cpu_count() or 1` before `_process_params("fit")` when user has not set it explicitly. Short-circuits LightGBM's joblib probe; honours an explicit user choice if already set.

**After (wall=3.2s, status=OK):**
- `_count_physical_cores_win32` / `subprocess.communicate` removed from top-30 hotspots.
- `lightgbm.basic.update` → 2.217s (was 2.337s; small library-internal improvement).
- **Wall 5.0s → 3.2s = 1.8s saved, ~36% speedup.**
