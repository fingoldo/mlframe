# Meta-test baselines

The `_*_baseline.json` files in this directory are frozen snapshots consumed by
the meta-tests (annotation coverage, bare-except inventory, debt-budget gates,
etc.). Each entry pins a known acceptance set so the meta-tests fail only on
new regressions, not on the historic backlog.

JSON does not allow inline comments, so this README is the canonical header for
every `_*_baseline.json` in this directory.

## How to regenerate

Regenerated by:

```
python tests/test_meta/regen_baselines.py
```

Run from the repository root. The script re-runs each meta-test in
"collect baseline" mode and rewrites the corresponding `_*_baseline.json`.

## Files

- `_annotation_baseline.json` - public-symbol annotation gaps allow-list
- `_bare_except_baseline.json` - bare `except:` locations allow-list
- `_console_unicode_baseline.json` - non-ASCII print/log call sites
- `_debt_baseline.json` - per-test deferred-debt budgets
- `_docstring_baseline.json` - missing-docstring allow-list
- `_logger_lazy_baseline.json` - non-lazy logger format strings
- `_mutable_defaults_baseline.json` - mutable default-argument sites
- `_resource_handle_baseline.json` - unclosed-resource sites
