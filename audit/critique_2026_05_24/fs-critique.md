# A1 Feature-selection critique of `train_mlframe_models_suite`

Scope: pre-screen, MRMR (filters), RFECV (wrappers), BorutaShap, registry resolver, and the suite-side pre_pipeline cache that the per-target loop relies on. Read-only pass; 15-minute wall budget. Findings ordered by severity, every issue surfaced (no top-N filtering).

## Summary

Train/val/test discipline is honest at the suite boundary: pre-screen is unsupervised + train-only by contract, FS only ever sees the train+val frames (test_df never enters `_apply_pre_pipeline_transforms`), and per-fold split inside RFECV preserves groups / time-series detection. The MRMR `_FIT_CACHE` key correctly folds `(_x_sig, _y_sig, _y_name, _y_full_hash, _params_sig)` and the cross-target identity cache defaults `mrmr_identity_cache_include_y=True`, so the canonical multi-target collision is closed. Real concerns that remain: the suite-side pre_pipeline cache `_content_fingerprint_for_cache` samples only 4 rows of `train_target`, balanced binary targets can collide; `general.estimate_features_relevancy` materialises an unconditional `arr.to_numpy(allow_copy=True).copy()` and mutates it with the legacy `np.random.shuffle` global RNG; MRMR accepts `groups=` but silently does NOT honour them in MI; RFECV `cv_n` inference at L140/L203 reads `self.cv` as a python int even when `self.cv` is already a splitter (subtle resize bug for callers passing a splitter directly); fleuret / permutation kernels still call `np.random.shuffle` on shared data without per-call RNG (parallel-fold determinism + thread-safety hazard); RFECV time-series auto-detect path silently honours a polars datetime hint even when the suite caller passes `cv=int` (silent strategy switch). No P0-grade leakage found.

## Не нашёл (искал и НЕ подтвердилось)

- **Target column visible in pre-screen** — pre_screen.py contract is unsupervised and `_phase_train_one_target_pre_screen.py:53-72` correctly protects every (target_type, target_name) pair across the suite + cat/text/embedding/group/timestamp columns. No target column gets dropped on a sibling-target's variance/null criterion. Earlier `if ctx.cat_features: pass` no-op is already fixed (L63).
- **MI/correlation computed on `train+val+test`** — `_apply_pre_pipeline_transforms` (_pipeline_helpers.py:413) fits on `train_df` and only `transform()` on `val_df`; test_df is never passed in. Confirmed by reading `_trainer_train_and_evaluate.py:389-403` and the call sites at trainer.py / _model_factories.py / _cb_pool.py — all four pass `train_df, val_df` only.
- **MRMR `_FIT_CACHE` cross-target collision** — key at `_mrmr_fit_impl.py:103` is `(_x_sig, _y_sig, _y_name, _y_full_hash, _params_sig)`, where `_y_full_hash = blake2b(np.ascontiguousarray(arr).tobytes(), digest_size=16)`. Full y content folded in; `_y_full_hash=""` triggers cache-skip (L100-101). Different targets cannot collide.
- **MRMR identity cache (`mrmr_skip_when_prior_was_identity=True`)** — default `mrmr_identity_cache_include_y=True` folds a 1000-cell y-sample blake2b into the cache key (`_mrmr_fingerprints.py:215`). Two distinct targets on the same X get separate slots by default. (The 6-decimal rounding at L215 is a separate sub-finding, see #10.)
- **RFECV groups support** — `_rfecv_cv_setup.py:119-134` correctly routes (classifier, groups) → `StratifiedGroupKFold`, (regressor, groups) → `GroupKFold`; the `_resolve_cv_and_val_cv` signature accepts `groups` explicitly and uses it for the dispatch.
- **RFECV TimeSeriesSplit support** — present and auto-detectable via four sources (polars-schema hint, fit_params timestamps hint, pandas DatetimeIndex, single polars datetime column). The over-eager auto-detect is documented separately as finding #6.
- **Test-set MI / leakage via cache cross-talk** — checked the pre_pipeline cache and pipeline-signature builder. Test_df is never threaded into either; no cross-iteration leakage of test rows.
- **MRMR re-runs per target with no sharing** — `mrmr_identity_cache` (per-ctx default, suite-scoped) AND `_FIT_CACHE` (process-scoped, LRU=4) both exist; cross-target reuse is in fact provided. Performance complaint not validated.
- **Polars-compat for FS** — MRMR has explicit polars branches throughout (`_mrmr_fit_impl.py:191-204`); RFECV converts polars → pandas at fit entry (line 102, intentional, see finding #12). Pre-screen handles both natively (`pre_screen.py:80-187`). Polars compat is real, not a degraded second-class path.
- **Group-aware ctx propagation** — `_trainer_train_and_evaluate.py:357-366` correctly slices `group_ids[train_idx]` before passing to pre_pipeline. No bug observed.
- **BorutaShap leakage** — `_boruta_shap_fit_explain.py:fit` operates only on the (X, y) it's given; the suite passes only train slices. No bug.
- **`signature == self.signature` cache shortcut** — both MRMR (`_mrmr_fit_impl.py:78-83`) and RFECV (`_rfecv_fit.py:257-265`) fold full y-content blake2b into the signature; same-shape-different-y replay is closed.
