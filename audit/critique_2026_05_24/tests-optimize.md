# tests/ optimization audit (B2) - 2026-05-24

## Summary

Test-infra audit on `mlframe/tests/` (8 conftests, 93 files using fast-mode markers/env, 196 session-scope fixtures across 86 files). Suite has solid plumbing - `--fast` flag, `fast_subset`, `no_xdist`/`fuzz` opt-outs, `cleanup_memory` fast-path on heavy markers, RFECV/tqdmu quiet monkey-patches with idempotency sentinels, session-scoped synthetic-data builders, numba JIT prewarm. Notable gaps: `@pytest.mark.fast` registered but ZERO test files apply it; only 2 files import `make_simple_*_data` (the rest re-roll synthetic data per file - duplication and fixture redundancy); a sizeable cluster of `pytest.skip("X not exported")` / `pytest.skip("MI screen happened to find features; fallback not exercised")` style swallows hide weak or stale tests; ZERO global `--timeout` in pyproject `addopts` (CI sets `--timeout=300`, local runs are unbounded); `_clear_mrmr_fit_cache_between_tests` is autouse but only relevant to a single test file. RFECV/tqdmu monkey-patches in `tests/conftest.py` point at leaky library defaults that should be fixed upstream. Two `importlib.reload` test files (`test_automl.py`, `test_phase_temporal_audit_polars_fastpath.py`) lack the snapshot/restore pattern mandated by `feedback_no_module_reload_without_snapshot`.
