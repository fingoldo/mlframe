# A3 Ensembling Critique — 2026-05-24

## Summary

OOF discipline is enforced at strict `score_ensemble` entry (gate + `max_ensembling_level>=2` raises on missing OOF) but the level-1 fallback in `_process_single_ensemble_method` silently substitutes in-sample `train_preds`/`train_probs` when OOF is unstamped, so the per-flavour "train" branch of every default level-1 ensemble can be evaluated on leaked rows whenever the trainer's `cross_val_predict` did not run. The ensemble-winner ranker (`_ENSEMBLE_RANK_METRIC_CANDIDATES`) silently falls back to `val.integral_error` and even `test.integral_error` despite the module's own `compare_ensembles` warning that the same val.* selection is a double-dip and test.* is a selection-bias trap. No probability calibration (Isotonic / Platt) is applied before any classifier blend. There is no `test_biz_val_ensembling*.py` quantifying that the ensemble actually beats the best single member, only shape and exclusion gates.

## Не нашёл

- A test asserting the level-2+ OOF guard at `_ensembling_score.py:228-239` actually raises (grep finds no `pytest.raises.*max_ensembling_level.*requires.*oof` in `tests/`).
- Per-class calibration of any kind (isotonic/Platt) anywhere on the score_ensemble path.

- A regression test for the `_finalize_per_target_ensembling` rrf_k metadata round-trip (train stamps rrf_k=10, predict reads rrf_k=10, blend output matches).
