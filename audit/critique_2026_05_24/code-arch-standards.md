# Code Architecture & Standards Audit — mlframe (2026-05-24)

NOTE: Persisted from inline agent output (Plan-type subagent ran without Write tool).

## Summary

mlframe enforces `--strict-markers --strict-config` yet ships at least one unregistered marker (`no_xdist_parallel` in `test_biz_val_pysr_fe_upgrade.py:17`) which trips strict-marker collection; pyproject also registers a dozen markers (`integration`, `multigpu`, `windows_only`, `linux_only`, `requires_xgb/lgb/cb/torch`, `uses_matplotlib`, `uses_torch`) that no test ever applies. Sklearn-compliance coverage is shallow: `tests/test_sklearn_compliance.py` calls only `check_is_fitted`/`hasattr`, never `sklearn.utils.estimator_checks.check_estimator`; `CompositeTargetEstimator`, `_LagPredictDeployableModel`, `ESTransformedTargetRegressor`, and the `RFECV` wrapper get no compliance harness. ~90 runtime deps lack upper bounds, including major-bump-prone polars, lightgbm, lightning, pydantic. Pre-commit runs detect-secrets + meta-tests only — no ruff/black/mypy/bandit gate. Docs corpus has not been updated alongside 2026-05-2x composite refactor.

## Архитектурное предложение

1. **Sklearn-compliance harness as gate, not afterthought.** Build `tests/sklearn_compliance/` with `parametrize_with_checks` over every estimator-shaped class (`CompositeTargetEstimator`, `_LagPredictDeployableModel`, `ESTransformedTargetRegressor`, `EstimatorWithEarlyStopping`, RFECV wrapper, RFECVRound if it ships) and run it in `sklearn-matrix-ci` so a sklearn minor never silently breaks the deploy path; today nothing prevents a `clone()`-induced data-loss like the one `_LagPredictDeployableModel` was specifically patched for.
2. **Tighten pyproject's deps + markers + lints contract.** Add upper-version caps on the dozen breaking-prone deps, delete twelve dead markers (or wire them in), register `no_xdist_parallel`, and move ruff/black/bandit from `continue-on-error` lint job into a hard pre-commit hook + blocking CI step. The "we have strict-markers" claim must hold true in practice.
3. **Pickle/cache RCE surface unification.** Centralise the `.sha256`-sidecar pattern from `inference/predict.py` into a single `mlframe.io.safe_pickle` module that the predict path, the composite cache, the FE cache, and any future `joblib.load` all go through; default-deny on missing sidecar with explicit `MLFRAME_ALLOW_UNVERIFIED_PICKLE=1` escape. Today there is one good gate and two unguarded `pickle.load` sites.
