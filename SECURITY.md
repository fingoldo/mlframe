# Security Policy

## Supported versions

mlframe is pre-1.0; security fixes are applied to the latest released version on PyPI and the `master` branch. Please upgrade to the latest version before reporting.

## Reporting a vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, email <fingoldo@gmail.com> with:

- A description of the vulnerability and its impact.
- Steps to reproduce (a minimal proof of concept if possible).
- The mlframe version (`python -c "import mlframe; print(mlframe.__version__)"`) and your environment.

You can expect an initial acknowledgement within a few days. Once the issue is confirmed and a fix is available, the fix will be released and the report disclosed responsibly with credit to the reporter (unless anonymity is requested).
